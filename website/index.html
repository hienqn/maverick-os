MaverickOS

What started as a UC Berkeley CS162 project turned into something bigger. I took PintOS and kept building on it—adding a write-ahead logging system for crash recovery, copy-on-write fork to make process creation faster, software RAID support (levels 0, 1, and 5), and even ported the whole thing to RISC-V.

Along the way, I replaced the old Perl testing infrastructure with TypeScript, added symbolic links, wrote VGA and PCI drivers, and implemented readers-writer locks. This site documents what I learned—real OS concepts tied to actual code, not just theory.

Beyond the Coursework

  • Write-ahead logging with UNDO/REDO recovery
  • Copy-on-write fork for lazy page copying
  • Software RAID (striping, mirroring, parity)
  • Symbolic links with loop detection

Dual Architecture

  • x86 (i386) as the primary target
  • RISC-V (rv64) with Sv39 MMU
  • VirtIO block drivers for RISC-V
  • Architecture-neutral abstractions

Modern Tooling

  • TypeScript test runner
  • Bun for fast builds
  • JSON test specifications
  • Parallel test execution
